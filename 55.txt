Введение
Наименование системы
Полное наименование системы: «Автоматизированная информационная система учета успеваемости» (далее — Система).
Назначение и цели создания
Система предназначена для автоматизации процессов контроля успеваемости студентов, ведения электронного документооборота кафедры и формирования аналитической отчетности.
Основными целями внедрения Системы являются:
Оптимизация работы преподавателя: переход от бумажных носителей к электронным форматам учета баллов за экзамены, дифференцированные зачеты и защиты проектов.
Прозрачность системы оценивания: создание единого цифрового пространства для мониторинга рейтинга студентов в режиме реального времени.
Систематизация пересдач: автоматизация формирования списков и ведомостей для повторных испытаний.
Область применения
Система будет использоваться сотрудниками учебной части, преподавательским составом и студентами в рамках образовательного процесса 
Краткое описание функциональности
В рамках проекта должна быть реализована возможность гибкого управления учебными группами, дисциплинами и баллами. Система должна поддерживать автоматический расчет итоговых сумм на основе текущего контроля, посещаемости и контрольных работ, а также обеспечивать вывод данных в печатные формы, соответствующие установленным образцам ведомостей.

Структура сайта (Эксклюзивно для Заместителей)
Сквозные элементы (Header и Footer)
В верхней части (Header) расположены:
Логотип: переход на общую панель управления (Дашборд).
Глобальный поиск: быстрый поиск по фамилии студента, номеру группы или названию дисциплины.
Меню разделов:
База рейтинга: ввод и редактирование текущих баллов (дата, дисциплина, баллы).
Генератор ведомостей: создание официальных бланков (Экзамен / Диф.зачет / Защита).
Реестр пересдач: управление списками задолжников.
Профиль Зама: настройки аккаунта, смена пароля, выход.
В нижней части (Footer):
Информационная панель: Общее количество групп в системе, дата последнего бэкапа данных.
Перечень страниц и функционал (Консоль Заместителей)	
Страница «Управление рейтингом» (Центральный ввод данных)
Так как заносят всё замы, им нужен максимально быстрый интерфейс:
Массовый ввод: выбор Группы + Дисциплины + Преподавателя → открывается таблица со всеми студентами.
Редактирование баллов: возможность быстро проставить баллы за посещаемость, КР и ДЗ (как на твоем фото-примере).
История правок: логирование того, какой именно Зам и когда изменил балл студенту.
Страница «Формирование ведомости»
Инструмент для официальной отчетности:
Тип контроля: выбор из списка (Экзамен, Диф. зачет, Защита проектов).
Автозаполнение: система сама подтягивает ФИО препода и дату из базы.
Результат: генерация чистого документа (PDF/Excel) со всеми рассчитанными оценками.
Раздел «Доп. сессия (Пересдачи)»
Автофильтр: кнопка «Показать всех с баллом < [X]».
Функционал: создание ведомости пересдачи в один клик для конкретной группы.
Страница «Справочники» (Настройки системы)
Группы: добавление/удаление учебных групп.
Преподаватели: список ФИО для подстановки в ведомости.
Дисциплины: перечень предметов по учебному плану.
Страница «Справочники» (Настройки системы) — Дополнение существующих пунктов 
Группы: наименование, курс, группа, список студентов 
Преподаватели: ФИО, направление программы (РПО или КГИД), индивидуальные примечания (окна доступности, к примеру: Асанов А.А все дни кроме пятницы, Потнин И.В. каждый день на 1 и 2 пары) 
Дисциплины: название, тип (лекция/практика), количество академических часов по учебному плану, связка «Дисциплина — Преподаватель(и)».
5. Новая страница «Генератор расписания» (Интеллектуальный модуль)
Интерфейс настройки: выбор периода (семестр/модуль/неделя).
Кнопка «Сгенерировать автоматически»: запуск алгоритма случайного распределения занятий с учетом заданных жестких и мягких ограничений.
Ручная корректировка: возможность перетаскивания пар в сетке расписания после автогенерации.
Контроль коллизий: автоматическая подсветка ошибок (например, если один преподаватель или группа случайно назначены в два места одновременно). 

Ключевые критерии и ограничения для алгоритма автогенерации:
Учет пожеланий преподавателей (Поле «Примечания»):
В карточке преподавателя реализуется матрица доступности (интерактивный календарь: Дни недели \(\times \) Номера пар).
«Жесткое ограничение» (Запрет): выбор дней и конкретных пар, в которые преподаватель физически не может присутствовать (например, среда — методический день, или только 1–2 пары). Алгоритм полностью исключает постановку занятий в это время.
«Мягкое ограничение» (Предпочтение): выбор приоритетных (удобных) дней и времени для ведения занятий. Алгоритм стремится в первую очередь заполнить эти слоты.
Базовые правила распределения (Алгоритм):
Случайный подбор (Рандомизация): базовое распределение пар строится случайным образом в рамках доступных временных окон для исключения предвзятости.
Неделимость нагрузки: система рассчитывает общую нагрузку (из справочника дисциплин) и равномерно распределяет её по неделям без превышения лимита пар в день для одной группы (не более 4 пар/8 академических часов).
Исключение «окон»: алгоритм минимизирует пустые перерывы между парами внутри одного дня как для студентов, так и для преподавателей. 

Логика навигации (User Flow для Зама)
Авторизация → 2. Выбор группы → 3. Ввод баллов за период → 4. Кнопка «Сгенерировать экзаменационную ведомость» → 5. Печать.
Требования к дизайну
3.1. Существующий сайт
Что не нравится в текущем процессе: ручной ввод данных в бумажные ведомости, риск потери данных, сложность подсчета итогового рейтинга вручную.
3.2. Пожелания к стилистике
Стиль: Строгий, минималистичный, структурный.
Характер: Сайт должен выглядеть как профессиональный рабочий инструмент (Dashboard). Никакой лишней анимации, ярких баннеров или мультипликации. Основной акцент — на читабельности таблиц и удобстве навигации.
Особенности: Крупные сетки, четкие границы таблиц, быстрый отклик интерфейса.
3.3. Цветовое решение
Основной фон: Светлый (белый или светло-серый #F5F7FA). Это снижает нагрузку на глаза при длительной работе с таблицами.
Фирменные цвета:
Фиолетовый — для бокового меню и заголовков.
Акцентный цвет (Синий или Спокойный зеленый) — для кнопок «Сохранить» или «Сформировать ведомость».
Цветовая индикация: Красный балл для задолжников, Зеленый — для успешно сдавших.

4. Технические треб	ования
Система управления и технологии
-
Интеграция со сторонним ПО
Экспорт данных: Обязательная интеграция с библиотеками для генерации файлов в форматах Excel (.xlsx) и PDF.
Печатные формы: Возможность прямой отправки сформированной ведомости на печать из браузера.
Авторизация: Система должна иметь закрытый вход с проверкой логина/пароля (учетные записи создаются только для Замов).
Адаптивность и устройства
Сайт предназначен в первую очередь для профессиональной работы за компьютером, так как ввод большого объема цифр неудобен с телефона.
Планшетная версия: Поддержка разрешения 1024 х 768 (горизонтальная ориентация) для возможности работы на выездных комиссиях.
Мобильная версия: Не требуется (или в режиме «только просмотр»), так как интерфейс широких таблиц не предназначен для малых экранов.
Браузерная поддержка
Корректная работа во всех актуальных версиях современных браузеров:
Google Chrome (последние 3 версии);
Apple Safari (12+);
Яндекс.Браузер (актуальная версия).
Безопасность и код
Стандарты: Код должен соответствовать стандартам W3C.
Безопасность: Защита от SQL-инъекций и XSS-атак.
Резервное копирование: Автоматическое создание бэкапа базы данных раз в сутки.
Логирование: Система должна записывать, какой пользователь (Зам) изменил оценку студенту (для исключения подлога данных).
